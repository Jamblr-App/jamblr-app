## What Is Jamblr?

Jamblr is an open-source, decentralized protocol for musicians to find each other, drop jam sessions on a live map, and connect without a middleman.

- 🗺 **Drop a pin** — broadcast a jam session to your local area
- 🔑 **Sovereign Identity** — your identity is a cryptographic key generated on your device. No account. No server. No one can take it from you.
- 🌊🔥🌍💨 **The Four Elements** — Water, Fire, Earth, Air. Every jam has a frequency. Find yours.
- ⚡ **Aether** — unlock the void when you've mastered all four elements

## How To Run It

Jamblr is a pure front-end web app. No server required.

1. Clone or download this repo
2. Open `index.html` in any modern browser
3. That's it

```bash
git clone https://github.com/Jamblr-App/jamblr-app.git
cd jamblr-app
# Open index.html in your browser
```

> **Note:** For full functionality (geolocation, map interactions), run it through a local server or use the live version at [jamblr.com](https://jamblr.com)

## Tech Stack

- **MapLibre GL** — open source maps, no Google, no tracking
- **Web Crypto API** — P-256 ECDSA key generation for sovereign identity
- **LocalStorage** — your data stays on your device
- **Vanilla JS / HTML / CSS** — no frameworks, no dependencies, no bullshit

## Philosophy

Jamblr is not a platform. Platforms extract value. Jamblr is a **protocol** — like HTTP, like BitTorrent. It routes signal between musicians and gets out of the way.

- No ads
- No algorithms deciding who gets seen
- No selling your data
- No account recovery (your key is your identity — keep it safe)
- No moderation (you are responsible for your own safety)

This is the red pill. If you want the blue pill, go back to sleep.
